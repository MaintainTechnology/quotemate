QuoteMate · Voice + SMS Agent Architecture

Two input channels → one shared intake + estimate pipeline → one customer SMS reply

Voice path

  • • Customer rings AU long code
  • • Vapi runs the live conversation (Jeff)
  • • Mid-call send_sms_photo_link tool fires upload SMS
  • • end-of-call-report webhook fires
  • • Transcript + photos persist in calls
  • • Hands off with callId

SMS path

  • • Customer texts AU long code +61 481 613 464
  • • Twilio → /api/sms/inbound (no Vapi)
  • • Haiku decides ask · finish · escalate (≤4 turns)
  • • First-turn intro + gratitude greeting
  • • Photo-upload SMS fires when job_type identified
  • • MMS attachments + upload-link photos both feed intake
  • • On finish, hands off with conversationId

Shared pipeline

  • • Opus 4.7 Vision structures the intake
  • • Quality gate: empty intakes get a callback SMS
  • • pgvector embedding + job_type pre-filter
  • • Voyage Rerank-2.5 re-scores top-20 → top-3
  • • Opus 4.7 + property-aware tools draft 3 tiers
  • • Semantic-grounding validator catches ungrounded prices
  • • Stripe Checkout sessions per tier (deposit only)

After the quote

  • • Quote SMS → customer (single-thread UX)
  • • Tradie Notify SMS + WhatsApp on every draft
  • • Customer taps link → /q/[token] HTML quote page
  • • Tier photos + render-time photo union (late uploads OK)
  • • Pays deposit → Stripe webhook → paid_at
  • • /q/[token]/book slot picker → confirmation SMS to both sides
  • • Daily cron sweeps abandoned conversations after 24h
