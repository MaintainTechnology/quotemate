Voice Agent · Recent quality upgrades

Voice agent is now hardened — quotes can't go off-rails.

Multiple defence layers shipped this week to ensure the AI cannot invent prices, hallucinate assemblies, or auto-quote risky scope. The agent's outcomes are now tightly bound to what's in our pricing book.

✓

Strict grounding on intake + estimation

System prompts now enforce that every line item in a quote must trace back to a row in shared_assemblies or shared_materials. The model literally cannot emit a price out of thin air.

✓

DB-grounding validator (4th defence layer)

Even if the model tries to slip past the prompt rules, a runtime validator checks every line item's source field against the database. Anything ungrounded is auto-downgraded to the $199 inspection route — no faulty quote reaches the customer.

✓

Quality gate on intake

Empty calls or calls with insufficient information no longer trigger downstream photo/quote SMS. Saves Twilio cost and avoids confusing customers with empty drafts.

✓

Opus 4.7 on intake structuring

Upgraded from Sonnet to Opus 4.7 for the IntakeSchema extraction step — meaningful quality gain on multi-job calls and ambiguous customer phrasing.

✓

S06 confidence routing live

Each draft is auto-classified HIGH / MED / LOW confidence based on intake completeness. HIGH → tradie sees a clean draft. LOW → automatic inspection upsell.

✓

Vapi prompt rebalanced

Receptionist tone improved — more natural, fewer interruptions, faster turn-taking. Customers report the call feels more human.

Quote Output · Customer-facing rendering

The output side — what the customer actually sees.

Per Jon's priority #5 — the agent now produces a real, viewable, mobile-friendly quote that the customer can read and pay a deposit on directly.

✓

Customer-facing quote page

Each draft has its own URL at /q/[id] — mobile-friendly HTML page showing scope of works, Good/Better/Best line items, exclusions, and assumptions. No PDF needed for v1; the link is what the customer sees in their SMS.

✓

Stripe Checkout deposit per tier

Each tier on the customer's quote page has its own "Pay deposit" button → goes to a tier-specific Stripe Checkout link. Webhook confirms payment. Thank-you and cancelled pages handle both flows.

✓

Inspection route — $199 pay link

For jobs flagged inspection-only (switchboard, EV charger, fault finding, complex renovations), the page shows a single $199 deposit option to book the inspection — same Stripe Checkout pattern.

✓

Photo upload — camera + gallery

Customer can attach photos via /upload route. Supports both phone camera capture and gallery picker. Photos feed into the intake structuring (Sonnet/Opus see them when extracting scope).

✓

WhatsApp fallback for SMS dispatch

When the customer-facing quote SMS fails to deliver (e.g. PH carrier blocks), the system falls back to WhatsApp via Twilio Sandbox. Already working for the existing voice-driven quote flow.

○

Mocked-up photos in quote

Showing example photos of the proposed work alongside Good/Better/Best (e.g. "tri-colour downlight" image next to that tier). Stock-image library keyed by job_type + tier — planned, not built yet.
