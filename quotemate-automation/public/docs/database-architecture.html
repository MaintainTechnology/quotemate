01 · The end-user view

Site features — which tables they touch

Each user-facing flow as it actually runs on production. Tables in monospace show the order they're read/written along the request path. This is the answer to "what gets touched when X happens?"

Feature / user journey Entry point Tables touched (in order)
Customer rings the Vapi number POST /api/vapi/webhook tenants (lookup by vapi_assistant_id) → calls (upsert by vapi_call_id, store transcript + recording) → triggers the intake pipeline below.
Customer texts the tradie's number POST /api/sms/inbound tenants (route by twilio_number) → sms_conversations (find-or-create, lock, update state) → sms_messages (idempotent insert by twilio_sid) → tenant_service_offerings + tenant_custom_assemblies + tenant_material_catalogue + shared_assemblies (AI dialog reads what tenant offers) → triggers intake pipeline.
Intake structuring (voice or SMS) POST /api/intake/structure calls + sms_messages + sms_conversations (gather raw input) → customers (find-or-create by phone, stamp tenant_id) → tenants (resolve trade + branding) → intakes (insert structured JSON + Voyage embedding).
Draft a quote POST /api/estimate/draft intakes + calls + sms_conversations (load context) → tenants + pricing_book (load tradie's rates) → shared_assemblies + shared_materials + tenant_custom_assemblies + tenant_material_catalogue + tenant_assembly_bom + tenant_assembly_overrides + tenant_service_offerings + tenant_material_preferences + tenant_tier_ladder (Opus tool-calls) → quotes (insert/update draft) → SMS dispatch.
Customer opens /q/[token] GET /q/[token] page quotes → intakes → tenants (branding) → pricing_book (licence footer) → sms_conversations (price-hold timer) → calls (display recording link if voice job).
Customer pays deposit via Stripe POST /api/stripe/webhook quotes (mark booking_state='reserved', stamp accepted_at) → tradies / tenants (notify the right inbox).
Customer picks a booking slot POST /api/q/[token]/book quotes + intakes + tradies → quotes (mark booking_state='booked', store slot).
Customer uploads photos POST /api/upload/[token] sms_conversations + quotes + calls + intakes (locate the job by token) → Supabase Storage intake-photos bucket → intakes (append paths to scope.photo_paths).
Tradie signs up POST /api/auth/signup → /onboard/* tradie_signup_intents (insert token) → email link → /onboard form → POST /api/onboard/activate → tenants (insert + provision Twilio/Vapi) → pricing_book (insert per-trade row) → tenant_service_offerings + tenant_licences (seed defaults) → /onboard/stripe for Connect Express.
Tradie opens the dashboard GET /dashboard GET /api/tenant/me → tenants + pricing_book + shared_materials + shared_assemblies + tenant_material_preferences + tenant_custom_assemblies + tenant_service_offerings + tenant_licences + calls + intakes + quotes + sms_conversations + sms_messages + payments. (This route is the dashboard's hot path — it reads almost everything.)
Tradie edits a quote tier POST /api/quote/[id]/edit quotes (load) → intakes + tenants + pricing_book (re-validate grounding) → quotes (update tier line items).
Tradie logs a follow-up touch POST /api/tenant/followups/* tenants + quotes + customers + intakes (resolve) → quote_followup_events (insert outcome+note) → optionally sms_conversations + sms_messages (send text/log call).
Tradie uploads supplier CSV POST /api/tenant/catalogue/upload tenants (auth) → tenant_material_catalogue (insert-only bulk) — provenance from supplier_catalogue if matched.
Operator runs admin trade-loader POST /api/admin/loader/upload admin_users (auth) → CSV parse → trades + categories + trade_pricing_defaults + trade_prompts + shared_assemblies + shared_materials + tenants (read for impact) → import_batches + import_staged_rows (staging) → approval → commit to live tables.
Cron: stale SMS cleanup GET /api/cron/sms-cleanup (Vercel cron) sms_conversations (close any conversation idle > 24h).
