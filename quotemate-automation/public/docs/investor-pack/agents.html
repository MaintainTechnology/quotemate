SMS Agent
Live in production · v1
Live

Answers every text to a tradie's number. Asks the right questions, captures photos, decides when it has enough, and hands off to the quote drafter — usually inside four turns.

How it thinks
  • Layer 1 — Intent. Regex first (80% of texts), Haiku 4.5 fallback for the tricky middle.
  • Layer 2 — Dialog. One Haiku call per turn. Returns: ask, finish, escalate-to-inspection, or end.
  • Layer 3 — Slot extraction. Pulls name, suburb, item count, specs from each reply.
  • Layer 4 — Handoff. On finish, fires intake structuring + quote drafting in the background.

What it knows about (today)
  • Electrical: downlights, GPOs, ceiling fans, smoke alarms, outdoor lighting
  • Plumbing: blocked drains, hot water, taps, toilets
  • Auto-escalates anything risky (gas leak, sparks, burst pipe) to a paid $99 inspection
  • Customer memory — returning customers skip name + address questions
Voice Agent
Provisioned per tradie · parked behind env flag
Ready

Answers phone calls to the tradie's number with the same brain. The full stack is built and assistants are spun up at signup — kept off in production today because voice costs (~$1,500/mo per tradie at moderate volume) don't fit SaaS pricing yet. v3 premium tier.

How it works
  • Speech in. Deepgram Nova-2, Australian English.
  • Dialog. Same Haiku 4.5 brain as SMS, same assumptions and rules.
  • Voice out. ElevenLabs (Australian male persona by default).
  • Mid-call tools. Can fire an SMS to the caller with a photo-upload link while still on the call.
  • Post-call. Transcript + photos flow into the same intake + quote drafter as SMS.

→Shared Brain Underneath

Both agents hand off to the same downstream pipeline. This is why adding the voice tier later is mostly a cost decision, not a build decision.

Intake Structuring

Claude Opus 4.7 reads the transcript + photos and writes a clean structured job brief. Strict grounding — only facts the customer actually said.

Quote Drafter

Opus 4.7 with tool-calling. Picks line items from the tradie's pricing book, builds three tiers, validates every price against the catalogue before saving.

Pricing Book

Per-tradie, per-trade. Hourly rate, markup, custom line items. The drafter never invents a price — it pulls from this and fails loudly if it can't.

Image Generator

Gemini renders a preview of each tier so the customer sees what they're buying. Product names piped through from the chosen catalogue item.
