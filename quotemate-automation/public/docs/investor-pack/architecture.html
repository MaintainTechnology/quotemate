Inbound
How the customer reaches us

Web Signup

▼
AI Brain
Decides what to do next
Intent Detection
Is this a customer or a tradie signing up? Regex first, Haiku if unclear
SMS Dialog Agent
One Haiku call per turn. Asks the next question or finishes.
Intake Structuring
Opus reads the transcript + photos, produces a clean job brief
Quote Drafter
Opus + tool-calling. Builds Good / Better / Best with real prices.
Image Generator
Gemini renders a preview of what each tier looks like installed
Slot Extractor
Pulls name, suburb, item count, specs from natural-language replies
▼
Data
Source of truth
Tenants
One row per tradie. Owns trades, licences, numbers, status.
Pricing Books
Per-tradie, per-trade rates and markup. Quotes only use these.
Shared Catalogue
Assemblies + materials for each trade (electrical, plumbing)
Conversations
Every SMS exchange, with photos, turn count, state
Intakes
Structured job briefs. Vector-embedded for similar-quote lookup.
Quotes
Three tiers, scope, deposit links, status (draft → sent → paid)
Customers
Persistent memory by phone. Returning customers skip questions.
▼
Delivery

Mobile Quote Page
/q/<token> — scope, photos, tiers, deposit buttons
Photo Upload
/upload/<token> — camera or gallery, one tap
Tradie Dashboard
Inbox, quotes, edit-in-place, retry provisioning
Tradie Notify SMS
Daily SMS + WhatsApp on every draft
Booking Page
Slot picker after deposit. Works, lightly exercised live.
