Commercial Paint · Estimator · Internal system note

A temporary file-store that makes every paint takeoff more accurate

mt-filestore-kb Gemini File Search Best-effort Self-cleaning Hybrid merge
After the estimator extracts a takeoff from a tradie's uploaded plans, this system spins up a private, throwaway knowledge store from those same documents, asks it to verify and fill the gaps, folds the answer back in under strict rules, then deletes itself. It only ever makes the existing extraction more correct — it never blocks a quote.
01 · Why it exists

The single extraction pass has blind spots

A commercial paint job arrives as a fat PDF plan set plus a painter's measurement sheet. One Claude Opus pass reads the plans and a Sonnet pass transcribes the measurements; the reconciler merges them into a takeoff. That works — but a single read of a 15-page set can miss a surface buried on page 9, mislabel a substrate, or carry a quantity the documents actually contradict.

↯

Missed surfaces

A soffit or stair core described only in a schedule note never makes it onto the takeoff — so it's never quoted.

≠

Wrong numbers

An area read off the wrong drawing, or a system inferred where the spec actually states something else.

?

Low-confidence gaps

Fields the model left blank or flagged unsure — height, substrate, coats — that pricing then has to guess at.

The fix isn't a bigger prompt. It's a grounded second opinion that can do retrieval over the full document set and cite the page it found each fact on.

02 · What it is

A per-job, throwaway knowledge store

The mt-filestore-kb service (a Gemini File Search wrapper) already powers QuoteMax's static trade-book knowledge. This system uses it differently: instead of a permanent library, it creates a temporary store from the tradie's own uploaded PDFs, queries it once to audit the takeoff, and tears it down. Nothing about the customer's documents persists in the KB after the run.

  • Grounded. Every answer is retrieved from the uploaded plans and must cite a page — it can't invent areas the documents don't show.
  • Private & ephemeral. The store is named per run, lives only for the extraction, and is deleted in a finally — including when something fails.
  • Best-effort. If the KB is unreachable, slow to index, or returns nonsense, the system silently falls back to the original takeoff. The quote never waits on it.

04 · The connections

QuoteMax ⇄ mt-filestore-kb ⇄ Gemini

Five HTTP calls over the existing typed client (lib/admin-loader/mt-filestore-kb.ts). The route never touches Gemini directly — the node brokers it. The teardown call (yellow) is guaranteed to run.

05 · The decision that protects the quote

Hybrid merge — fill gaps, flag conflicts

The takeoff feeds pricing, so the KB is never allowed to silently overwrite a number a customer is quoted on. applyPaintSupplement() is a pure function with three outcomes. A confident value the documents disagree with becomes a flag for the tradie, not a rewrite.

⊕

kb_filled

The field was empty or the line was low-confidence. The KB value is filled in and the line is noted.

⚑

kb_conflict

The line was already confident and the documents disagree. The value is kept; a suggested correction is flagged for review.

＋

kb_added

The documents describe a paintable surface absent from the takeoff. It's appended as a low-confidence line for the tradie to confirm.

06 · The temporary store's life

Born, used, deleted — in one run

The store exists only inside supplementTakeoffViaKb(). The teardown sits in a finally, so the store is removed on the happy path, on a search error, and on an indexing timeout alike. Nothing leaks.

01

Create & load

A run-scoped store is created and the uploaded PDFs are streamed in for chunking + embedding.

02

Query once

After indexing settles, a single grounded search audits the takeoff and returns strict JSON findings.

03

Destroy

finally → kbDeleteStore(force:true). A cleanup failure is swallowed; it can never surface as an extraction error.

  • Never throws. Any failure returns the original takeoff with usedKb:false.
  • Deterministic to test. Network, sleep and the poll budget are injected, so the orchestrator is unit-tested with no live calls.
  • Gated. Runs only when the KB env is present and PAINT_KB_SUPPLEMENT_ENABLED ≠ false — one switch disables it everywhere.
07 · How it helps

Tighter takeoffs, no new risk

Fewer missed lines

Surfaces hidden in schedules or later pages get surfaced and added — so they're priced, not lost.

Gaps filled with evidence

Blank or unsure fields are completed from the documents, each carrying a page citation the tradie can check.

Less manual correction

The tradie opens the editor to a cleaner takeoff with conflicts already flagged, instead of re-reading the plans cold.

The quote is protected

Confident, customer-facing numbers are never silently changed; the money path keeps its human-in-the-loop discipline.
