Onboarding Bundle Spec · v1
Priority #4 · Trade onboarding

Each new trade ships as an onboarding bundle.

Electrical is the reference. To add a second trade (plumbing, painting, HVAC) we ship one bundle of five components — the same five every time. This doc defines the shape so the next trade's onboarding is mechanical, not a re-design.

The five components.

A bundle is the smallest unit that lets the SMS Agent + Voice Agent quote a new trade end-to-end.

1. Base assemblies

The pre-priced building blocks the Estimation Engine looks up to construct G/B/B tiers (e.g. "LED downlight assembly").

2. Per-job intake rules

For each "easy 5" job the trade can auto-quote: safe defaults, must-ask fields, inspection triggers.

3. Default pricing

Hourly rate, callout minimum, GST flag, markup %, risk-buffer %. Tradie overlays during onboarding.

4. G/B/B framing

For each easy-5 job, what the customer expects to see in Good vs Better vs Best (warranty, brand, smart features).

5. Licence schema

Per-state regulatory display: NSW NECA, VIC ESV, QLD QBCC. Customer-facing footer on every quote.

Shape of a bundle.

A bundle is a single TypeScript module exporting five named pieces. The SMS Agent and Voice Agent both import from this module — no other code changes per trade.

File layout

lib/trades/<trade>/
├── index.ts                // re-exports the bundle
├── assemblies.ts           // 1. base assemblies (component #1)
├── intake-rules.ts         // 2. per-job rules + universal triggers (component #2)
├── pricing-defaults.ts     // 3. hourly rate, GST, markup, risk-buffer (component #3)
├── tier-framing.ts         // 4. customer-facing G/B/B label hints (component #4)
└── licence-schema.ts       // 5. per-state licence fields + display (component #5)

The bundle interface (TypeScript)

export interface TradeBundle {
  trade: 'electrical' | 'plumbing' | 'painting' | 'hvac'
  easyFive: ReadonlyArray<JobType>       // jobs the SMS Agent will auto-quote
  inspectionOnly: ReadonlyArray<JobType> // jobs that route to $99 site visit

  // 1. assemblies — seeded into shared_assemblies + shared_materials
  assemblies: ReadonlyArray<AssemblySeed>

  // 2. intake rules — consumed by lib/sms/dialog.ts SYSTEM_PROMPT
  intakeRules: Record<JobType, AssumptionRule>
  universalInspectionTriggers: ReadonlyArray<string>
  universalMustAsk: ReadonlyArray<string> // always: name, suburb, job_type

  // 3. pricing defaults — seeded into pricing_book
  pricingDefaults: PricingBookSeed

  // 4. tier framing — used by Estimation prompt + tier-photos
  tierFraming: Record<JobType, { good: string; better: string; best: string }>

  // 5. licence schema — per-state regulator display
  licenceSchema: Record<'NSW'|'VIC'|'QLD'|'WA'|'SA'|'TAS'|'ACT'|'NT', LicenceDisplay>
}

Worked example — adding plumbing.

Concrete preview of what the plumbing bundle would look like. Numbers and rules are illustrative — finalised with the pilot plumber.

Plumbing easy 5 (auto-quote)

  • tap_replacement — replace existing kitchen / bathroom tap, like-for-like
  • toilet_replacement — swap an existing toilet, no new plumbing
  • shower_head — replace shower head / arm, existing pipe
  • basin_replacement — replace bathroom basin, existing waste + supply
  • laundry_tap — replace laundry tap or washing-machine valve

Plumbing inspection-only (always $99 site visit)

  • hot water system — new or replacement (sizing + gas/electric routing)
  • drainage / blocked sewer line
  • leak detection and repair
  • renovation rough-in
  • backflow prevention installation

Universal inspection triggers (plumbing)

Mostly different from electrical — different danger signals. Replace the existing UNIVERSAL_INSPECTION_TRIGGERS with the trade's own:

  • burst pipe, flooding, water damage, ceiling stain
  • no hot water, gas smell, water hammer
  • backflow, sewage smell, drain backup
  • asbestos pipework, lead pipework, pre-1970 plumbing

Pricing defaults (illustrative)

  • Hourly rate: $120/hr
  • Callout minimum: $130
  • Markup: 22%
  • Licence: NSW Master Plumbers Association (or NSW Fair Trading plumber's licence)

G/B/B framing (tap_replacement example)

  • GOOD: Standard chrome mixer, 1yr warranty
  • BETTER: WELS 5-star ceramic-disc mixer, 5yr warranty
  • BEST: Premium designer mixer, 10yr warranty, lead-free brass
Build with the tradie, not for the tradie

Per CLAUDE.md: most owner-operators don't have a structured price list. Ship the bundle skeleton above, then capture the pilot plumber's overlay through a guided onboarding screen (their actual rates, their preferred brands, their warranty positions). The bundle is the shape; the tradie fills in the numbers.

What stays trade-agnostic.

The whole pipeline below the bundle never changes when you add a trade. This is the leverage.

Files you DO touch when adding a trade

  1. Create lib/trades/<trade>/ with the five files
  2. Add the trade's seed SQL: sql/seed/<trade>_assemblies.sql
  3. Add the trade's job_type values to IntakeSchema in lib/intake/schema.ts
  4. Add the trade's JOB_TYPE_LABEL entries in lib/sms/templates.ts + app/q/[token]/page.tsx
  5. Add the trade's tier descriptions to lib/quote/tier-photos.ts
  6. Wire the bundle into the dialog SYSTEM_PROMPT (one line that picks the active trade's rules)

That's it. Six edits, one new directory, no architectural changes.

Onboarding the tradie themselves.

Once the bundle is in code, the new tradie's individual onboarding (their actual rates, brands, warranty positions) is a guided UI flow.

Six-step tradie onboarding (planned)

  1. Identity + licence — ABN, licence number, state, photo of licence card
  2. Default pricing overlay — adjust hourly, callout, markup; Stripe Connect Express setup
  3. Tier framing review — accept default G/B/B framing for each easy-5 job, or substitute their own brand preferences
  4. Assumption rules walkthrough — review each job's safeDefaults + mustAsk; correct anything that doesn't match how they work
  5. Test conversation — simulate one SMS and one voice intake; tradie sees the exact draft quote that will go to a customer; corrects anything wrong
  6. Phone number provisioning — Twilio number for SMS + Vapi number for voice (both AU long codes)
Critical — eval before live customers

Per CLAUDE.md: 100 hold-out (intake → quote) pairs scored on a 5-dimension rubric. No tradie goes live without their bundle achieving threshold scores on the eval set, customised with their corrected pricing.
