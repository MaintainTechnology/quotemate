# Per-tenant File Store — Spec

## Objective
Give every QuoteMate tenant (tradie) their own dedicated, durable file store: a downloadable PDF archive in Supabase Storage plus a per-tenant Gemini File Search store for search/Q&A. At the moment any quote PDF is finalized/sent — across all live trades — and whenever a tenant uploads an invoice, the system (a) archives the *full, unredacted* customer document to Supabase Storage and (b) ingests a *server-generated, PII-minimized text rendering* of that document into the tenant's Gemini store. A one-time backfill seeds existing active tenants; everything new flows automatically thereafter. This unlocks (P2) a Files tab to browse/download the archive and a cited KB chat, and (P3) grounding new quote drafts on the tenant's own past jobs and invoices.

The split is deliberate and is the core privacy mechanism: **Gemini never receives a customer-facing PDF or a raw invoice image.** It only ever receives a minimized text/markdown doc with customer name / street address / phone / email stripped or masked. The durable, access-controlled source of truth for the real document lives only in Supabase Storage and is never sent to Gemini.

**Three hard API constraints that shape the whole design:**
1. **Gemini File Search does NOT persist raw files** — only chunks + embeddings; there is no endpoint to download an original back. The durable, downloadable copy lives ONLY in Supabase Storage. **Consequence:** KB citations reference the minimized text doc's `displayName`, not page numbers of the customer PDF; the Supabase archive remains the source of truth, and the P2 download path always streams from Supabase, never from the KB.
2. **Single global `KB_API_KEY`, no per-tenant auth** — every store is reachable with that one key. Tenant isolation is 100% QuoteMate's job: the tenant→store mapping is server-side only and a store id is never sent to a client.
3. **Upload ≤100MB; async indexing 10–60s** (states `processing → active → failed`); accepts PDF/Office/text/markdown/html/csv/json, skips audio/video/archives/exe. **There is no synchronous "indexing done" signal** — confirmation is poll-only via `kbListDocuments` (the reconcile cron, R15).

### Per-document source-id and displayName conventions (resolves prior ambiguity)
A concrete builder must never guess the key. Fixed rules:

| Trade / kind | `tenant_file_documents.source_id` value | `display_name` value | Notes |
|---|---|---|---|
| electrical / plumbing (quote) | `quote.id` (uuid) | `quote-<trade>-<quote.id>` | `quotes` is the only row type keyed by quote id |
| roofing (quote) | `roofing_measurements.public_token` | `quote-roofing-<public_token>` | no `quotes` row exists for roofing |
| solar (quote) | `solar_estimates.public_token` | `quote-solar-<public_token>` | **the in-scope `estimate` object exposes `.token`/`.public_token`, NOT `.id`** — use the token; do not attempt `estimate.id` |
| residential / commercial painting (quote) | `painting_measurements.public_token` | `quote-painting-<public_token>` | commercial save-quote also stores PDF at `quotes/<quoteId>.pdf`, but the **tracking source_id is still the painting public_token** for that trade |
| invoice | `invoice_uploads.id` (uuid) | `invoice-<invoice_uploads.id>` | |

**Rule of thumb baked into `archiveAndIngestQuote`:** `source_kind='quote'` uses `quote.id` only for electrical/plumbing; **all token-based trades (roofing, solar, painting) use `public_token`**; `source_kind='invoice'` uses `invoice_uploads.id`. `source_id` is `text` precisely so it holds either a uuid or a token. All display names are extension-free (the ingested doc is `.md`).

The Supabase full-doc archive is **retained** (it is the source of truth and cheap); only the KB index is pruned. Pruning is logged per tenant. (Long-term paid-storage decision past the free tier remains an Open Question.)
    - **Material re-draft handling (resolved, NEW):** when a quote at the same id/token is re-ingested (via the edit hook R9 or re-run) and its `content_hash` **differs** from the stored row, the helper **replaces** the stale KB doc (`kbDeleteDocument` old `kb_document_id`, then upload new, update `content_hash`) rather than dedup-skipping. Identical `content_hash` → dedup-skip as before. This closes the "stale doc kept on material re-draft" gap.

- **Reuse fidelity (hard):** `slug`/`MAX_DISPLAY_NAME` in `store-name.ts` are module-private today and **must be `export`ed in this PR** before `tenant-store-name.ts` imports them; `bareStoreId` is already exported. `app/api/tenant/calibration/upload/route.ts` is fully synchronous today and **must have `after()` added** for invoice archive+ingest — it is not extending an existing `after()`.
- **KB API:** single global `KB_API_KEY` (isolation 100% app-layer); KB holds chunks+embeddings only (no raw download); upload ≤100MB (`KB_UPLOAD_MAX_BYTES`); async indexing 10–60s, states `processing→active→failed`, confirmation poll-only via `kbListDocuments`; ingested docs always `text/markdown`; `DELETE` on a non-empty store needs `?force=true`.
- **Env:** reuse `loadKbConfigFromEnv` (`KB_API_URL ?? MT_FILESTORE_KB_URL`, `KB_API_KEY ?? MT_FILESTORE_KB_API_KEY`; throws if missing). - **Isolation:** tenant→`file_store_id` mapping server-side only; store id never sent to a client; P2 list/download/chat and P3 grounding scoped by authenticated `tenant_id`; deterministic store name keyed on tenant UUID.
- **Next 16:** per `quotemate-automation/AGENTS.md`, read the relevant `node_modules/next/dist/docs/` guide before writing Next code. Webhook/finalize routes fast-ack <500ms; all KB ingest runs post-ack via `next/server` `after()`.
- **Failure isolation:** every new path best-effort, NEVER throws into or blocks the quote/invoice pipeline (mirrors `addDocumentsToSessionStore`).
- **Migration discipline:** new `134_tenant_file_store.sql` (+ `134_down.sql`) + `scripts/run-migration-134.mjs` applied to prod Supabase; keep `sql/init.sql` representative.
- **Idempotency:** dedup by `displayName` + UNIQUE `(tenant_id, display_name)`; deterministic display names so re-send/re-backfill never duplicate; **material re-drafts (differing `content_hash`) replace the stale KB doc rather than skip.**
- **Cost/retention:** Gemini indexing ~$0.15/1M tokens (once at ingest); minimized markdown ≪ a PDF, so per-doc cost is low. KB store bounded by R15(c) pruning at `TENANT_FILESTORE_MAX_DOCS`/free-tier; Supabase full-doc archive retained as source of truth.

## Edge cases to handle
- **Indexing slow (10–60s):** ingest marks `state='pending'`, returns immediately; reconcile cron flips to `active` once `kbListDocuments` shows it ready.
- **Indexing returns `failed`:** row `state='failed'` + `error`; reconcile retries while `attempts < TENANT_FILESTORE_MAX_RETRIES`, incrementing `attempts` each pass, then stays `failed`.
- **Empty `kbText`:** helper no-ops with a log line; no KB upload, no `active` row.
- **`file_store_id` null at finalize:** helper lazily `ensureTenantStore`, writes the id, then ingests.
- **Store missing/deleted upstream:** label-tolerant find-or-create recreates it; on a 404 upload, re-ensure once then give up best-effort.
- **`tenant_id IS NULL` (orphan/test rows):** helper no-ops + log; backfill skips entirely.
- **Re-sent quote (identical content, same id/token):** deterministic `displayName` + matching `content_hash` → dedup-skip; UNIQUE prevents a duplicate row.
- **Materially re-drafted/re-priced quote (same id/token, different `content_hash`):** helper deletes the stale KB doc and uploads the new one (R15), updates `content_hash`; the tracking row is updated in place (UNIQUE preserved).
- **Re-run backfill / `--resume`:** zero new KB docs and zero new tracking rows for already-`active` rows.
- **Residential-painting backfill (lazy PDF / null `pdf_path`):** backfill **eagerly calls `ensurePaintingPdf(public_token)` first**, then ingests; if `ensurePaintingPdf` returns null (Gotenberg down / inspection), skip + log.
- **KB API down / `KbHttpError`:** full PDF still lands in Supabase; KB step caught + logged; row left `pending`; reconcile picks it up. Quote/invoice flow unaffected.
- **Gotenberg down (`ensure*Pdf` null):** no full doc → helper no-ops + log (lockstep: no ingest without archive); quote still sends.
- **Force-confirm / inspection-routed:** price-hidden PDF → archive it + ingest `pricesHidden:true` matching text; `needs_inspection`/null PDF → no-op. Never ingest a price-bearing variant.
- **Invoice is an image (JPG/PNG/HEIC):** raw image archived to `invoices/<id>.<ext>`; only the minimized extracted text ingested — image never sent to Gemini.
- **PII leak via minimizer regression:** correctness bug; a unit test (DoD) asserts no name/street/phone/email substring survives minimization.
- **Store exceeds `TENANT_FILESTORE_MAX_DOCS` / free tier:** reconcile prunes oldest `active` KB docs via `kbDeleteDocument`, marks rows `skip_reason='pruned'`, retains the Supabase archive.
- **Flag off:** all paths STUB; no KB calls, no rows, no behavior change.
- **Cross-tenant access attempt (P2):** download/chat for a doc whose `tenant_id` ≠ caller → 404; never leak the other store's id.
